    software en específico

    Existen

    API NATIVAS
    API EXTERNAS

REST (REPRESENTATIONAL STATE TRANSFER)
    Un servicio rest es un conjunto de restricciones
    con las que podemos crear un estilo de 
    arquitectura de software

    Caracteristicas importantes

    1. Relación cliente servidor: el cliente y el 
    servidor están debilmente acoplados es decir
    estan separados.

    2. Las API REST son sin estado: es decir que
    cada petición recibida por el servidor se va
    a manejar de manera independiente, lo que evita
    que mantenga sesiones conectadas como para hacer
    varias peticiones al mismo tiempo.

    3. Debe se ser cacheable: debe de haber un
    sistema de almacenamiento en caché para evitar
    repitivas conexiones hacia un mismo recurso del
    servidor y esto impacta en el redmiento de las
    aplicaciones.

    4. Hay que mantener una interfaz uniforme: 
    esto quiere decir una interfaz generica que
    permita la administración entre la interacción
    entre el cliente y el servidor de una manera
    uniforme. Es decir que para cada petición va
    a existir una URL única y con las Caracteristicas
    de como nosotros le vamos a enviar la información,
    a esto se le conoce como ENDPOINTS

    5. Sistemas de capas: esto significa que del lado
    del servidor debe de haber varias capas a la hora
    de implementarla para estar previstos a sí más 
    adelante ese desarrollo escala.
